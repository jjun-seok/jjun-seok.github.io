# 1. SELECT문

> 테이블에서 데이터를 조회하는 가장 기본적인 명령어

```sql
-- 문제 1. albums 테이블에서 앨범 제목만 조회
SELECT Title FROM albums;

-- 문제 2. artists 테이블의 모든 정보 조회
SELECT * FROM artists;

-- 문제 3. tracks 테이블 앞 10건만 조회
SELECT * FROM tracks LIMIT 10;

-- 문제 4. tracks 테이블에서 곡 이름과 단가만 조회 (앞 10건)
SELECT Name, UnitPrice FROM tracks LIMIT 10;
```

---

# 2. DISTINCT — 중복 제거

> 중복된 값을 제거하고 고유한 값만 반환

```sql
-- 문제 1. 고객 나라 목록 (중복 없이)
SELECT DISTINCT Country FROM customers;

-- 문제 2. 곡 단가 종류 (중복 없이)
SELECT DISTINCT UnitPrice FROM tracks;

-- 문제 3. 도시 + 나라 조합 (중복 없이)
SELECT DISTINCT City, Country FROM customers;
```

---

# 3. WHERE + 비교 연산자

> 조건에 맞는 행만 필터링

| 연산자 | 의미 | 예시 |
|--------|------|------|
| `=` | 같다 | `Country = 'USA'` |
| `>` `<` | 크다/작다 | `UnitPrice > 0.99` |
| `>=` `<=` | 이상/이하 | `Total >= 10` |
| `!=` | 다르다 | `Country != 'USA'` |

```sql
-- 문제 1. 미국 고객만 조회
SELECT * FROM customers WHERE Country = 'USA';

-- 문제 2. 단가 0.99 초과인 곡 (앞 10건)
SELECT * FROM tracks WHERE UnitPrice > 0.99 LIMIT 10;

-- 문제 3. 결제 금액 10 이상인 인보이스
SELECT * FROM invoices WHERE Total >= 10;
```

---

# 4. 복합 조건 (AND / OR / NOT)

| 연산자 | 의미 |
|--------|------|
| `AND` | 두 조건 **모두** 참 |
| `OR` | 두 조건 중 **하나라도** 참 |
| `NOT` | 조건을 **반전** |

```sql
-- 문제 1. 미국 + 캘리포니아 고객
SELECT * FROM customers WHERE Country = 'USA' AND State = 'CA';

-- 문제 2. 미국 또는 캐나다 고객
SELECT * FROM customers WHERE Country = 'USA' OR Country = 'Canada';

-- 문제 3. 미국이 아닌 고객
SELECT * FROM customers WHERE NOT Country = 'USA';
```

---

# 5. BETWEEN / IN / NOT IN

```sql
-- BETWEEN: 범위 조건 (경계값 포함)
-- 문제 1. 결제 금액 5~10달러 사이
SELECT * FROM invoices WHERE Total BETWEEN 5 AND 10;

-- IN: 여러 값 중 하나와 일치
-- 문제 2. 미국, 캐나다, 브라질 고객
SELECT * FROM customers WHERE Country IN ('USA', 'Canada', 'Brazil');

-- NOT IN: 여러 값 모두 제외
-- 문제 3. 미국, 캐나다, 브라질 제외 고객
SELECT * FROM customers WHERE Country NOT IN ('USA', 'Canada', 'Brazil');
```

# 6. LIKE — 유사한 값 찾기

| 패턴 | 의미 |
|------|------|
| `'Rock%'` | Rock으로 **시작** |
| `'%Live'` | Live로 **끝남** |
| `'%Love%'` | Love가 **포함** |

```sql
-- 문제 1. 'Rock'으로 시작하는 앨범
SELECT * FROM albums WHERE Title LIKE 'Rock%';

-- 문제 2. 'Live'로 끝나는 앨범
SELECT * FROM albums WHERE Title LIKE '%Live';

-- 문제 3. 'Love'가 포함된 곡 (앞 10건)
SELECT * FROM tracks WHERE Name LIKE '%Love%' LIMIT 10;
```

---

# 7. ORDER BY — 정렬

```sql
-- 문제 1. 가장 비싼 곡부터 (내림차순, 앞 10건)
SELECT * FROM tracks ORDER BY UnitPrice DESC LIMIT 10;

-- 문제 2. 결제 금액 가장 큰 인보이스 5건
SELECT * FROM invoices ORDER BY Total DESC LIMIT 5;

-- 문제 3. 앨범 제목 알파벳 오름차순 (앞 10건)
SELECT * FROM albums ORDER BY Title ASC LIMIT 10;
```

# 8. INSERT — 데이터 삽입

```sql
-- 문제 1. 컬럼명 명시하여 삽입
INSERT INTO book (id, title, category, price, rating, stock)
VALUES (13, 'It''s Only the Himalayas', 'Travel', 45.17, 2, 19);

-- 문제 2. 컬럼명 생략 (테이블 정의 순서대로)
INSERT INTO book
VALUES (14, 'Set Me Free', 'Young Adult', 17.46, 5, 19);

SELECT * FROM book;
```

---

# 9. UPDATE — 데이터 수정

> ⚠️ WHERE 없이 UPDATE하면 **전체 행**이 수정됨!

```sql
-- 문제 1. 특정 책 가격 변경
UPDATE book SET price = 39.99 WHERE title = 'Soumission';

SELECT * FROM book;

-- 문제 2. Poetry 카테고리 재고 일괄 +5
UPDATE book SET stock = stock + 5 WHERE category = 'Poetry';

SELECT * FROM book;
```

---

# 10. DELETE — 데이터 삭제

> ⚠️ WHERE 없이 DELETE하면 **전체 행**이 삭제됨!

```sql
-- 문제 1. 평점 1점인 책 삭제
DELETE FROM book WHERE rating = 1;

SELECT * FROM book;

-- 문제 2. 재고 16권 미만인 책 삭제
DELETE FROM book WHERE stock < 16;

SELECT * FROM book;
```

# 핵심 정리

| 명령어 | 용도 |
|--------|------|
| `SELECT` | 데이터 조회 |
| `DISTINCT` | 중복 제거 |
| `WHERE` | 조건 필터링 |
| `AND/OR/NOT` | 복합 조건 |
| `BETWEEN` | 범위 조건 |
| `IN/NOT IN` | 목록 조건 |
| `LIKE` | 패턴 매칭 |
| `ORDER BY` | 정렬 |
| `INSERT` | 데이터 삽입 |
| `UPDATE` | 데이터 수정 |
| `DELETE` | 데이터 삭제 |

> ⚠️ UPDATE / DELETE 에서 **WHERE 빠지면 전체 데이터** 수정·삭제됨. 항상 주의!
