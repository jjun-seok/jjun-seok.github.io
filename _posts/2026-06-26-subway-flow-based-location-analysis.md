저는 팀에서 **데이터 분석 파트**를 맡았고, 이 글에서는 진행 과정과 분석 결과를 정리합니다.

### 핵심 처리 — 환승역 합산

원본 데이터는 같은 역이라도 호선마다 행이 따로 기록되어 있었습니다. 예를 들어 "잠실"은 2호선 행, 8호선 행이 따로 존재했습니다. 이걸 그대로 두면 환승역의 유동인구가 실제보다 낮게 집계되는 문제가 생깁니다.

```python
def clean_name(name):
    return name.split('(')[0].strip()

result['역명_clean'] = result['지하철역'].apply(clean_name)

station_level = result.groupby('역명_clean').agg(
    평균승차인원=('평균 승차 인원', 'sum'),
    평균하차인원=('평균 하차 인원', 'sum'),
    호선목록=('호선명', lambda x: ', '.join(sorted(set(x))))
).reset_index()
```

역명을 정규화한 뒤 그룹화해서 합산하니, **순위 자체가 바뀌었습니다.** 단일 호선만 봤을 때는 강남이 1위였지만, 환승역을 합산한 역 단위 기준에서는 **잠실이 1위**로 올라왔습니다. 단순히 "사람이 많은 역"이 아니라 "여러 노선의 흐름이 겹치는 역"이 실제로는 더 큰 유동인구 풀을 갖고 있다는 뜻입니다.

## 최종 결과 — 월총유동인구 Top10

시간대 평균값만 보면 직관적으로 와닿지 않아서, 시간대별 인원을 **합계**로 환산한 월총유동인구 지표를 추가로 계산했습니다.

| 순위 | 역 | 호선 | 월총유동인구 |
|---|---|---|---|
| 1 | 잠실 | 2·8호선 | 약 443만 명 |
| 2 | 강남 | 2호선 | 약 438만 명 |
| 3 | 고속터미널 | 3·7·9호선 | 약 394만 명 |
| 4 | 서울역 | 1·4호선 등 5개 노선 | 약 378만 명 |
| 5 | 홍대입구 | 2호선·경의선·공항철도 | 약 374만 명 |
| 6 | 선릉 | 2호선·분당선 | 약 347만 명 |
| 7 | 신림 | 2호선 | 약 335만 명 |

상위권에 환승역(잠실, 고속터미널, 서울역, 사당)이 다수 포진해 있는 게 눈에 띕니다.

### 역의 성격은 승하차 차이로 갈린다

같은 Top10 역이라도 사람의 흐름 방향은 정반대일 수 있습니다.

- **승차 우세(+)** → 사람들이 "타고 떠나는" 역. 귀가·생활권 성격 (강남, 신림, 사당)
- **하차 우세(−)** → 사람들이 "내려서 모이는" 역. 업무·상업·관광 성격 (잠실, 홍대입구, 가산디지털단지)
- **균형(±0 근접)** → 환승 거점 성격 (고속터미널, 선릉, 서울역)

같은 "유동인구가 많은 역"이라도, 강남과 잠실은 정반대 성격을 갖고 있습니다. 이건 같은 상품이라도 역마다 판매 전략이 달라져야 한다는 뜻이기도 합니다.

### 지도 시각화

역 위치좌표 데이터를 병합해서, 서울 전체 520개 역을 지도 위에 원으로 표시했습니다. 원의 크기는 총유동인구 규모에 비례하고, Top10 역은 빨간색으로 강조했습니다.

```python
for _, row in merged.iterrows():
    is_top10 = row['역명_clean'] in top10_names
    color = '#E74C3C' if is_top10 else '#3498DB'
    folium.CircleMarker(
        location=[row['x좌표'], row['y좌표']],
        radius=max(3, (row['총유동인구'] + 1) / 4000),
        color=color, fill=True, fill_color=color, fill_opacity=0.6
    ).add_to(map_osm)
```

지도를 직접 확대해보면, Top10 역(빨간 원)이 2호선·7호선 라인을 중심으로 한강 이남과 서울 도심에 집중되어 있는 패턴을 확인할 수 있었습니다.

## 팀 프로젝트로 확장 — 입지에서 상품까지

데이터 분석은 여기서 끝이 아니라, 팀 프로젝트의 출발점이었습니다. 팀에서는 이 분석 결과를 바탕으로:

1. **역 유형 분류** — 승하차 패턴 + 시간대 피크를 결합해 역을 여러 유형(출근형/퇴근형/상권형/업무지구형 등)으로 분류
2. **소비 트렌드 결합** — 2015~2021년 소비 흐름(코로나기 위생·간편식 트렌드 포함)을 상품 추천에 반영
3. **역별 상품 전략 매핑** — "잠실엔 위생용품+간식", "강남엔 커피+우산" 같은 역별 맞춤 패키지 제안

까지 이어지는 컨설팅 보고서를 완성했습니다. 데이터 하나(지하철 유동인구)가 입지를 결정하고, 또 다른 데이터(소비 트렌드)가 상품을 결정하는 구조로, 두 축이 합쳐져야 "팔릴 자리"가 보인다는 게 팀이 잡은 핵심 메시지였습니다.

## 배운 점

- **합계 처리의 중요성**: 환승역을 제대로 합산하지 않으면 분석 결과 자체가 왜곡될 수 있다는 걸 직접 확인했습니다.
- **지표 표현 방식의 차이**: 같은 데이터라도 "시간당 평균"과 "월 합계"는 전달력이 완전히 다릅니다. "시간당 18만 명"보다 "한 달 443만 명"이 훨씬 직관적으로 다가옵니다.
- **숫자 하나가 곧 결론은 아니다**: 유동인구 1위(잠실)와 2위(강남)는 숫자는 비슷하지만 흐름의 성격이 정반대였습니다. 데이터를 다각도로 쪼개봐야 진짜 인사이트가 나온다는 걸 배웠습니다.
