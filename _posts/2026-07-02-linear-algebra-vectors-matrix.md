## 왜 선형대수학을 배우는가

머신러닝·딥러닝은 결국 숫자 더미를 다루는 일이고, 그 숫자 더미를 효율적으로 표현·조작하는 도구가 **선형대수학**이다. 문장은 단어 벡터(BoW)로, 이미지는 픽셀 행렬로, 신경망의 한 층은 `u = Wx + b`라는 행렬-벡터 연산으로 표현된다. 즉 선형대수를 모르면 "모델 안에서 실제로 무슨 계산이 일어나는지"를 이해할 방법이 없다.

```python
# 신경망 1층 계산 예시: u = Wx + b, y = ReLU(u)
import numpy as np

x_input = np.array([[1.0], [0.5], [-0.3]])       # 입력 (3×1)
W = np.random.randn(4, 3)                          # 가중치 (4×3)
b = np.random.randn(4, 1)                           # 편향 (4×1)
u = W @ x_input + b                                 # 선형결합
y_relu = np.maximum(0, u)                           # 활성화 함수
```

이 강의 커리큘럼은 **기초(벡터·행렬 연산) → 심화(벡터공간·SVD·고유값) → ML 응용(PCA·확률·미분) → DL 응용(역전파)** 순으로 짜여있다. 이번 글은 그중 기초 파트를 정리한다.

## Part 1. 벡터 — 크기와 방향을 가진 양

### 벡터란 무엇인가

벡터는 **크기(magnitude)**와 **방향(direction)**을 동시에 가진 양이다. 시작점이 달라도 크기·방향이 같으면 같은 벡터로 취급한다 — 단, **회전 이동**해서 겹치는 건 같다고 보지 않는다. 이게 "같은 물리량인데 위치만 다르다"는 벡터 특유의 성질이다.

### 벡터공간 $R^n$

$R^n$은 실수 n개의 순서쌍 집합에 **덧셈**과 **스칼라 곱** 두 연산을 정의한 것이다.

```python
a = np.array([1, 2])
b = np.array([3, -1])
a + b          # 성분별 덧셈 → [4, 1]
3 * a          # 스칼라 곱 → [3, 6]
```

이 연산은 교환법칙, 결합법칙, 분배법칙 같은 공리를 만족한다 — 이걸 일일이 외울 필요는 없고, "벡터 연산은 우리가 아는 산수 규칙을 그대로 따른다"는 것만 기억하면 된다.

> 벡터는 1D 배열(`shape=(3,)`)로도, 2D 열벡터(`shape=(3,1)`)로도 표현 가능하다. `row_vec.reshape(-1, 1)`로 행벡터를 열벡터로 바꿀 수 있는데, 행렬 연산에서 이 shape 차이 때문에 에러가 자주 난다. `(3,)`와 `(3,1)`은 다른 shape라는 걸 꼭 기억해두자.
{: .prompt-warning }

### 내적(Inner Product)과 노름(Norm)

**내적**은 두 벡터의 성분별 곱의 합이고, **노름**은 벡터 자기 자신과의 내적에 루트를 씌운 것 — 즉 벡터의 "크기"다.

```python
x = np.array([1, -1, 2])
y = np.array([-1, 0, 2])

inner = np.dot(x, y)          # 내적: 1×(-1) + (-1)×0 + 2×2 = 3
norm_x = np.linalg.norm(x)    # L2 노름(유클리디안): sqrt(1+1+4)
dist = np.linalg.norm(x - y)  # 두 벡터 사이 거리
```

거리·노름은 $p$ 값에 따라 세 가지로 나뉜다.

| 종류 | 공식 | 별명 |
|---|---|---|
| L1 | $\sum \lvert x_i - y_i \rvert$ | 맨해튼 거리 |
| L2 | $\sqrt{\sum (x_i-y_i)^2}$ | 유클리디안 거리 (기본값) |
| L∞ | $\max \lvert x_i - y_i \rvert$ | 최댓값 거리 |

```python
np.linalg.norm(x - y, ord=1)      # L1
np.linalg.norm(x - y, ord=2)      # L2 (기본)
np.linalg.norm(x - y, ord=np.inf) # L∞
```

### 두 벡터 사이의 각도 — 코사인 유사도

내적을 노름으로 나누면 두 벡터 사이의 각도(코사인 값)가 나온다.

$$\cos\theta = \frac{\langle v, w \rangle}{\|v\| \|w\|}$$

```python
def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
```

이게 바로 추천 시스템·NLP에서 쓰는 **코사인 유사도**다. 두 벡터의 방향이 비슷할수록 1에 가깝고, 수직(내적=0)이면 0, 반대 방향이면 -1이 나온다. 크기(길이)는 무시하고 순수하게 "방향이 얼마나 비슷한가"만 보는 지표라서, 문서 임베딩처럼 벡터 길이가 문서 길이에 따라 들쭉날쭉할 때 특히 유용하다.

> **Cauchy-Schwarz 부등식**($\lvert\langle v,w\rangle\rvert \leq \|v\|\|w\|$)은 코사인 값이 항상 -1~1 사이에 있다는 걸 수학적으로 보장해준다. 그래서 `np.arccos()`에 넣기 전에 부동소수점 오차로 범위를 살짝 벗어나는 걸 `np.clip(cos_theta, -1, 1)`로 방지하는 게 관례다.
{: .prompt-tip }

## Part 2. 행렬 — 벡터들의 집합

### 행렬의 정의

m×n 행렬은 행이 m개, 열이 n개인 숫자 배열이다. 행과 열의 수가 같으면 **정방행렬**, 1×n이면 행벡터, n×1이면 열벡터라고 부른다.

```python
A = np.array([[1, 2, -1],
              [3, 2, -1]])   # 2×3 행렬
print(A.shape)               # (2, 3) → 2행 3열
print(A[0, 0])                # a_11 성분: 1
```

### 기본 연산 — 같은 크기끼리만

덧셈·뺄셈은 **같은 크기**의 행렬끼리만 가능하고, 성분별로 계산된다. 스칼라 곱은 모든 성분에 곱한다.

```python
A + B    # 크기가 다르면 에러
3 * A    # 스칼라 곱
```

### 행렬 곱셈 — 여기서 실수가 제일 많이 남

$$A_{(m \times r)} \cdot B_{(r \times n)} = C_{(m \times n)}$$

핵심 조건 하나: **앞 행렬의 열 수 = 뒤 행렬의 행 수**. 이게 안 맞으면 애초에 곱셈 자체가 정의되지 않는다.

```python
A = np.array([[5, 6], [7, 8]])   # 2×2
B = np.array([[1, 2], [3, 4]])   # 2×2

A @ B     # 행렬 곱 (정상)
A * B     # 원소별 곱 — 행렬 곱과 다른 연산! 헷갈리면 안 됨
```

> `AB ≠ BA`. 행렬 곱셈은 **교환법칙이 성립하지 않는다.** 순서를 바꾸면 결과가 완전히 달라지거나 아예 곱셈이 성립하지 않을 수도 있다. 신경망 레이어 순서가 중요한 이유이기도 하다.
{: .prompt-warning }

### 전치행렬과 대칭행렬

전치($A^\top$)는 행과 열을 맞바꾸는 연산이다. 전치해도 자기 자신과 같으면 **대칭행렬**이라고 부른다.

```python
A = np.array([[1, 2, 3], [4, 5, 6]])
A.T                          # (2,3) → (3,2)

S = np.array([[2, 4, 0], [4, 1, 6], [0, 6, 3]])
np.allclose(S, S.T)          # True → 대칭행렬
```

### 단위행렬과 역행렬

**단위행렬** $I$는 대각선만 1이고 나머지는 0인 정방행렬로, 곱해도 원래 행렬이 그대로 나온다($AI=IA=A$) — 숫자에서 1과 같은 역할이다.

**역행렬** $A^{-1}$은 $AA^{-1}=I$를 만족하는 행렬이다. 2×2 행렬은 공식으로 바로 구할 수 있다.

$$A^{-1} = \frac{1}{ad-bc}\begin{pmatrix} d & -b \\ -c & a \end{pmatrix}$$

```python
A = np.array([[2., 5.], [-3., -7.]])
A_inv = np.linalg.inv(A)
np.allclose(A @ A_inv, np.eye(2))   # True
```

### 행렬식(det) — 역행렬이 존재하는지 알려주는 신호

$$\det(A) = ad - bc \quad (2\times2)$$

**det(A) = 0이면 역행렬이 존재하지 않는다.** 이런 행렬을 특이행렬(singular matrix)이라고 부른다.

```python
def inv_2x2(A):
    d = A[0,0]*A[1,1] - A[0,1]*A[1,0]   # 오류 상황: det=0이면
    if abs(d) < 1e-10:
        raise ValueError('역행렬이 존재하지 않습니다 (det=0)')  # 여기서 막아야 함
    return (1/d) * np.array([[A[1,1], -A[0,1]], [-A[1,0], A[0,0]]])
```

### 선형방정식 풀기 — Gauss-Jordan 소거법

연립방정식 $Ax=b$는 이론적으로는 $x = A^{-1}b$로 풀 수 있지만, 실무에서는 역행렬을 직접 구하는 대신 **Gauss-Jordan 소거법**으로 첨가행렬 $[A|b]$를 기약행 사다리꼴(RREF)로 바꿔서 푸는 게 수치적으로 더 안정적이다.

```python
# 직접 구현 대신 실무에서는 이렇게 푼다
A = np.array([[1, 2, -1], [4, 9, -3], [-2, -3, 7]])
b = np.array([1, 8, 10])
x = np.linalg.solve(A, b)   # Ax=b를 바로 풀어줌
```

## 오늘의 핵심 정리

| 연산 | A 크기 | B 크기 | 결과 크기 | 조건 |
|---|---|---|---|---|
| A + B | m×n | m×n | m×n | 크기 같아야 함 |
| λA | m×n | - | m×n | λ는 스칼라 |
| AB | m×r | r×n | m×n | A의 열 수 = B의 행 수 |
| $A^\top$ | m×n | - | n×m | - |
| $A^{-1}$ | n×n | - | n×n | det(A) ≠ 0 |

1. 벡터 = 크기 + 방향, 벡터공간 $R^n$은 이 벡터들에 덧셈·스칼라곱 연산을 정의한 집합.
2. 내적으로 각도(코사인 유사도)까지 구할 수 있다 — 방향 유사성 측정의 기본 도구.
3. 행렬 곱은 `AB ≠ BA`, `*`은 원소별 곱이지 행렬 곱이 아니다 — 이 둘을 헷갈리면 바로 버그.
4. 역행렬은 `det(A) ≠ 0`일 때만 존재하고, 실무에서 연립방정식은 역행렬을 직접 구하기보다 `np.linalg.solve()`로 푸는 게 일반적.
