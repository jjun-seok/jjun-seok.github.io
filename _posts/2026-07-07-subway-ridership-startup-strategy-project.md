## 프로젝트 한 줄 소개

팀명 **짭켄지 & Co.**로 진행한 1차 미니 프로젝트다. **2021년 6월 서울 지하철 승하차 데이터**를 갖고 "소자본 예비창업자가 어느 역에서 무엇을 팔면 좋을까"라는 질문에 데이터로 답을 찾아보는 분석 프로젝트다.

> ⚠️ 실제 무허가 노점 운영을 권장하는 자료가 아니다. 컨설팅 보고서 형식을 빌려온 **풍자적 사고실험**이고, 목적은 "지하철 데이터를 실제 의사결정 형태로 번역해보는 과정"을 보여주는 데 있다.

---

## 왜 이 질문이 흥미로운가

일반적인 지하철 분석은 "어느 역이 제일 붐비나?" 정도로 끝난다. 이 프로젝트는 여기서 한 발 더 들어가서, "사람이 많이 지나는 역"을 "소자본 판매자가 만날 수 있는 잠재 고객 흐름"으로 재해석했다. 핵심 메시지는 이거다.

> **사람 많은 곳이 아니라, 돈 꺼낼 사람이 흐르는 곳을 찾아야 한다.**

단순히 유동인구 1위 역을 찾는 게 아니라, **승차·하차 흐름의 방향**으로 역의 성격을 해석하고, 그 성격에 맞는 상품군까지 제안하는 게 이 프로젝트의 핵심이다.

---

## 데이터와 분석 절차

- **데이터**: 서울시 공공데이터포털의 지하철 호선별·역별·시간대별 승하차 인원 정보
- **분석 기준**: 2021년 6월, 전체 호선 통합 약 520개 역
- **처리 절차**: 6월 데이터 추출 → 환승역 역명 정규화·합산 → 역별 평균 승차/하차/총유동인구·승하차차이 계산 → Top10 도출

### 핵심 지표 정의

| 지표 | 의미 |
|---|---|
| 총유동인구 | 평균 승차 인원 + 평균 하차 인원 |
| 승하차차이 | 평균 승차 인원 − 평균 하차 인원 |

**승하차차이**가 이 프로젝트의 핵심 아이디어다. 양수(+)면 "타고 떠나는" 사람이 많은 **주거·귀가형** 역, 음수(−)면 "내려서 모이는" 사람이 많은 **업무·상업형** 역으로 해석했다.

## 코드로 보는 핵심 로직

### 1. 환승역 합산 — 역명 정규화

잠실처럼 2호선·8호선으로 나뉘어 집계된 환승역을 하나로 합치지 않으면 유동인구가 실제보다 적게 잡힌다.

```python
# "잠실(송파구청)" -> "잠실"처럼 괄호 뒷부분을 잘라내서 역명을 통일
def clean_name(name):
    return name.split('(')[0].strip()

result['역명_clean'] = result['지하철역'].apply(clean_name)

station_level = result.groupby('역명_clean').agg(
    평균승차인원=('평균 승차 인원', 'sum'),
    평균하차인원=('평균 하차 인원', 'sum'),
    호선목록=('호선명', lambda x: ', '.join(sorted(set(x))))
).reset_index()

station_level['총유동인구'] = station_level['평균승차인원'] + station_level['평균하차인원']
```

### 2. 시간대별 평균 계산

```python
on_cols = [c for c in metro_recent.columns if '승차인원' in c]
off_cols = [c for c in metro_recent.columns if '하차인원' in c]

result['평균 승차 인원'] = metro_recent[on_cols].mean(axis=1)  # 24개 시간대 평균
result['평균 하차 인원'] = metro_recent[off_cols].mean(axis=1)
result['승하차차이'] = result['평균 승차 인원'] - result['평균 하차 인원']
```

### 3. 지도 시각화 — folium

```python
import folium

top10_names = set(top10_total['역명_clean'])
map_osm = folium.Map(location=[37.5665, 126.9780], zoom_start=11, tiles='cartodbpositron')

for _, row in merged.iterrows():
    is_top10 = row['역명_clean'] in top10_names
    radius = max(3, (row['총유동인구'] + 1) / 4000)   # 유동인구에 비례한 원 크기
    color = '#E74C3C' if is_top10 else '#3498DB'       # Top10은 빨간색으로 강조
    folium.CircleMarker(
        location=[row['y좌표'], row['x좌표']],
        radius=radius,
        color=color,
        fill=True,
        popup=f"{row['역명_clean']} - 총유동인구 {row['총유동인구']:.0f}명"
    ).add_to(map_osm)
```

각 역의 유동인구를 원의 크기로 표시하고, Top10 역만 빨간색으로 강조해서 한눈에 "어디가 몰려있나"를 보여줬다.

## 주요 발견

**승차 1위는 강남(92,076명)인데, 총유동인구(승차+하차) 1위는 잠실(184,414명)**이었다. 승차만 보면 강남이 1위지만, 승차·하차를 합치면 환승역인 잠실이 앞선다 — "사람이 많은 역"과 "여러 노선이 겹치는 역"은 다르다는 걸 보여주는 지점이다.

### 승하차 흐름으로 나눈 역 유형 3가지

| 유형 | 조건 | 해석 | 예시 |
|---|---|---|---|
| 귀가·생활권형 | 승차 > 하차 | 타고 떠나는 사람이 많음 | 강남, 신림, 사당 |
| 유입·출근형 | 하차 > 승차 | 내려서 모이는 사람이 많음 | 잠실, 홍대입구, 가산디지털단지 |
| 환승·균형형 | 승차 ≈ 하차 | 타고 내리는 사람이 비슷함 | 선릉, 고속터미널, 서울역 |

### 소비 트렌드와 결합한 상품 전략

2020~2021년 코로나 시기 소비 트렌드(위생·간편식·생활밀착)를 반영해서, 역 유형별로 추천 상품군을 매칭했다. 예를 들어 유입형인 잠실·서울역은 마스크·손소독제 같은 즉시 필요 상품, 귀가형인 신림은 저가 생활소모품이 적합하다고 봤다.

## 데이터로 말할 수 있는 것과 가정한 것을 구분하기

이 프로젝트에서 제일 신경 쓴 부분이다. 데이터가 직접 보여주는 사실과, 거기서 해석·가정한 부분을 섞지 않고 명확히 나눴다.

- **데이터로 직접 확인한 것**: 역별 승차/하차/총유동인구, Top10 순위, 승차 우세/하차 우세 여부
- **데이터에서 해석한 것**: "하차 우세 역은 외부 유입이 강하다" 같은 패턴 해석
- **별도 가정이 포함된 것**: 연령대별 소비 특성, 역별 추천 상품군 — 승하차 데이터에 없는 정보라 일반적인 소비 트렌드를 근거로 한 전략적 가정임을 명시했다.

## 한계점

- 2021년 6월 기준 데이터라 현재 상권·유동인구와 차이가 있을 수 있음
- 코로나 시기 데이터라 일반적인 이동 패턴과 다를 수 있음
- 유동인구 수치일 뿐 실제 구매 전환율은 포함하지 않음
- 날씨, 경쟁자, 단속 리스크 같은 현실 변수는 반영하지 않음

---

## 오늘의 핵심 정리

1. 환승역은 호선별로 쪼개져 있으므로 역명 정규화 후 합산해야 실제 유동인구가 나온다.
2. 승차와 하차를 따로 보면 "역의 성격"(귀가형/유입형/균형형)을 해석할 수 있다.
3. 유동인구 총량만 보지 말고 흐름의 방향까지 봐야 상품 전략을 제대로 짤 수 있다.
4. 데이터로 확인한 사실과, 그 위에 얹은 가정을 구분해서 보고서를 쓰는 게 설득력 있는 분석의 기본이다.
